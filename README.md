# Lucky Draw

누구나 쉽게 만드는 나만의 럭키드로우 이벤트 플랫폼

생일 카페, 팬미팅, 프로모션 등 다양한 이벤트에서 사용할 수 있는 웹 기반 럭키드로우 서비스입니다.
텍스트 모양으로 배치된 카드를 터치하면 1~6등 결과가 나옵니다.

---

## 주요 기능

- **자유로운 컨셉 설정** — 이벤트 제목, 로고, 배경 이미지를 직접 커스터마이징
- **텍스트 카드 배치** — 입력한 텍스트(한글/영문/숫자) 모양으로 카드가 배치됩니다
- **카드 뽑기 애니메이션** — 카드를 터치하면 뒤집히며 등수 표시 + 1~3등 축하 컨페티
- **실시간 모니터링** — 운영자 페이지에서 추첨 현황을 실시간 확인
- **다중 이벤트 지원** — 한 계정으로 여러 이벤트를 독립적으로 운영
- **모바일 최적화** — 스마트폰에서 바로 사용 가능

## 시작하기

### 사전 준비

- [Node.js](https://nodejs.org/) 18 이상
- [MySQL](https://www.mysql.com/) 5.7 이상

### 설치

```bash
git clone https://github.com/your-username/lucky-draw.git
cd lucky-draw
npm install
```

### 환경 설정

`.env.example`을 복사하여 `.env` 파일을 만듭니다.

```bash
cp .env.example .env
```

`.env` 파일을 열고 아래 항목을 설정합니다.

```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=비밀번호
DB_NAME=birthday_draw
PORT=3000
JWT_SECRET=
AES_KEY=
```

**JWT_SECRET과 AES_KEY 생성:**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

이 명령어를 **2번** 실행하여 각각 `JWT_SECRET`과 `AES_KEY`에 붙여넣으세요.

### 서버 실행

```bash
npm start
```

서버가 시작되면 DB와 테이블이 자동으로 생성됩니다.
`http://localhost:3000` 에서 접속할 수 있습니다.

## 사용 방법

### 1단계: 회원가입 & 로그인

1. `http://localhost:3000/signup` 에서 이메일, 비밀번호, 이름을 입력하여 가입
2. 가입 완료 후 자동으로 대시보드로 이동

### 2단계: 이벤트 만들기

1. 대시보드에서 **"+ 새 이벤트 만들기"** 클릭
2. 이벤트 제목 입력 (예: "생일 럭키드로우", "팬미팅 이벤트")
3. 생성 후 자동으로 이벤트 관리 페이지로 이동

### 3단계: 이벤트 설정

이벤트 관리 페이지에서 다음을 설정합니다.

#### 뽑기판 텍스트

텍스트를 입력하면 각 글자의 픽셀 모양대로 카드가 배치됩니다.

| 입력 예시 | 결과 |
|-----------|------|
| `LUCKY` | 1개 탭, "LUCKY" 모양 카드 |
| `HAPPY DAY` | 2개 탭 ("HAPPY" / "DAY") |
| `지우 생일축하` | 2개 탭 ("지우" / "생일축하") |
| `해원 225` | 2개 탭, 한글+숫자 혼용 |

- 한글, 영문(A-Z), 숫자(0-9), 공백 사용 가능
- **공백으로 탭(페이지)이 구분됩니다**

#### 경품 배분

**자동 배분** — 카드 수에 따라 아래 비율로 자동 분배됩니다.

| 등수 | 비율 |
|------|------|
| 1등 | 1% (최소 1개 보장) |
| 2등 | 2% |
| 3등 | 7% |
| 4등 | 10% |
| 5등 | 20% |
| 6등 | 60% |

**수동 배분** — 각 등수별 수량을 직접 지정합니다. 합계가 총 카드 수와 일치해야 합니다.

#### 뽑기판 색상

색상 피커 또는 HEX 코드(`#RRGGBB`)로 카드 앞면 색상을 변경할 수 있습니다.

#### 로고 & 배경 이미지

- **로고**: 참여자 페이지 상단에 표시 (최대 2MB, jpeg/png/gif/webp)
- **배경**: 참여자 페이지 전체 배경 (최대 5MB)

### 4단계: 카드 생성 & 이벤트 시작

1. 뽑기판 텍스트 설정 완료 후 **"카드 생성"** 클릭
2. 이벤트 상태를 **"active(진행중)"**으로 변경
3. **참가 링크 복사** 버튼으로 참여자에게 공유할 URL 획득

### 5단계: 참여자에게 카드 뽑기

1. 참여자가 공유받은 링크(`/event/이벤트코드`)에 접속
2. 이벤트 제목과 로고가 표시됩니다
3. 카드를 터치하면 뒤집히면서 등수가 표시됩니다
4. 1~3등은 축하 컨페티 효과가 나옵니다

### 6단계: 실시간 모니터링

운영자 이벤트 관리 페이지에서 확인 가능:

- **Summary** — 전체 / 추첨 완료 / 잔여 카드 수
- **Prize Status** — 등수별 추첨 현황
- **Draw History** — 추첨 이력 (시간, 위치, 페이지, 등수)

### 7단계: 이벤트 종료

- 이벤트 상태를 **"closed(종료)"**로 변경하면 참여자 페이지에 "종료된 이벤트입니다" 표시
- **데이터 리셋** 버튼으로 카드 데이터를 삭제하고 새로 시작할 수 있습니다

## 배포

### HTTPS 설정 (권장)

공유 Wi-Fi 환경(카페 등)에서 사용한다면 HTTPS를 권장합니다.

**Caddy (가장 간단):**
```bash
caddy reverse-proxy --from yourdomain.com --to localhost:3000
```

**로컬 네트워크 전용이면 ngrok:**
```bash
ngrok http 3000
```

리버스 프록시 사용 시 `server.js`에 아래 설정을 추가하세요:
```javascript
app.set('trust proxy', 1);
```

## 문제 해결

### "JWT_SECRET이 설정되지 않았습니다"
`.env` 파일에 `JWT_SECRET` 값이 비어있습니다. 위의 생성 명령어로 값을 설정하세요.

### "이미 사용 중인 이메일입니다"
동일한 이메일로 가입한 계정이 이미 존재합니다. 로그인 페이지에서 기존 계정으로 로그인하세요.

### "이미 생성된 뽑기판이 있습니다"
카드를 새로 생성하려면 이벤트 관리 페이지에서 먼저 **데이터 리셋**을 진행하세요.

### 카드가 안 보이는 경우
1. 이벤트 관리 페이지에서 카드 생성이 완료되었는지 확인
2. 이벤트 상태가 **"active(진행중)"**인지 확인

### DB 연결 실패
`.env`의 `DB_HOST`, `DB_USER`, `DB_PASSWORD` 값이 올바른지, MySQL 서버가 실행 중인지 확인하세요.
